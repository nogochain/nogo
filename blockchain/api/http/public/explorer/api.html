NogoChain API Reference

✅ Production-Ready API

Complete RESTful API for interacting with NogoChain blockchain. All endpoints return JSON responses. Base URL: http://localhost:8080

📝 Authentication

Most endpoints are public. Admin endpoints require X-Admin-Token header with valid miner address.

Chain Information

GET /chain/info

Get comprehensive blockchain information including height, supply, difficulty, and network stats.

Response Example

{ "version": "1.0.0", "chainId": 1, "height": 12345, "latestHash": "abc123...", "genesisHash": "def456...", "totalSupply": 100000000000000000, "peersCount": 50, "difficultyBits": 123456789, "blockReward": 5000000000, "nextHalving": 525600 }
GET /chain/special_addresses

Get special system addresses (Community Fund, Integrity Pool, Genesis).

Response Example

{ "communityFund": { "address": "NOGO00...", "balance": 50000000000000 }, "integrityPool": { "address": "NOGO01...", "balance": 10000000000000 }, "genesis": { "address": "NOGO02...", "balance": 100000000000000 } }
GET /version

Get node version and build information.

Blocks

GET /block/height/{height}

Get block by height with full transaction details.

Parameters

Name Type Description
height uint64 Block height (path parameter)

Response Example

{ "height": 12345, "hash": "abc123...", "prevHash": "def456...", "timestampUnix": 1234567890, "difficultyBits": 123456789, "nonce": 987654321, "minerAddress": "NOGO00...", "transactions": [...], "coinbase": { "minerReward": 4800000000, "fee": 200000000, "data": "block reward: community=100000000, genesis=50000000, integrity=50000000" } }
GET /block/hash/{hash}

Get block by hash with full transaction details.

GET /blocks/from/{height}

Get multiple blocks starting from a height (pagination supported).

Query Parameters

Name Type Description
limit int Number of blocks to return (default: 10, max: 100)
GET /headers/from/{height}

Get block headers only (lighter than full blocks) from a height.

POST /block

Submit a new block to the network (for miners).

Authentication Required

Requires X-Admin-Token header with valid miner address.

Request Body

{ "height": 12345, "prevHash": "abc123...", "timestampUnix": 1234567890, "difficultyBits": 123456789, "nonce": 987654321, "minerAddress": "NOGO00...", "transactions": [...], "totalWork": "1234567890" }

Response

{ "accepted": true, "reorged": false }

Transactions

GET /tx/{txid}

Get transaction details by transaction ID.

Response Example

{ "transaction": { "type": "transfer", "fromPubKey": "base64...", "toAddress": "NOGO00...", "amount": 1000000000, "fee": 1000000, "nonce": 5, "data": "", "signature": "base64..." }, "location": { "height": 12345, "blockHashHex": "abc123...", "index": 2 }, "txId": "txid123..." }
POST /tx

Broadcast a signed transaction to the network.

Request Body

{ "type": "transfer", "fromPubKey": "base64...", "toAddress": "NOGO00...", "amount": 1000000000, "fee": 1000000, "nonce": 5, "signature": "base64..." }

Response

{ "txId": "abc123...", "accepted": true }
POST /tx/batch

Broadcast multiple transactions in a single request.

Request Body

{ "transactions": [ { "type": "transfer", ... }, { "type": "transfer", ... } ] }

Response

{ "results": [ { "txId": "abc...", "accepted": true }, { "txId": "def...", "accepted": true } ] }
GET /tx/receipt/{txid}

Get transaction receipt and execution status.

GET /tx/status/{txid}

Get transaction status (pending/confirmed).

Addresses

GET /balance/{address}

Get account balance and nonce for an address.

Response Example

{ "address": "NOGO00...", "balance": 10000000000, "nonce": 42 }
GET /address/{address}/txs

Get transaction history for an address with pagination.

Query Parameters

Name Type Description
limit int Number of transactions (default: 20, max: 100)
offset int Pagination offset

Response Example

{ "address": "NOGO00...", "balance": 10000000000, "nonce": 42, "transactions": [ { "txId": "abc123...", "type": "transfer", "amount": 1000000000, "timestamp": 1234567890, "direction": "in" } ], "total": 150, "limit": 20, "offset": 0 }

Mempool

GET /mempool

Get current mempool status and pending transactions.

Response Example

{ "size": 150, "txs": [ { "txId": "abc123...", "type": "transfer", "amount": 1000000000, "fee": 1000000, "timestamp": 1234567890 } ] }

Mining

GET /mining/info

Get current mining information and network difficulty.

Response Example

{ "difficulty": 123456789, "difficultyBits": 123456789, "networkHashrate": "1.5 TH/s", "blockReward": 5000000000, "nextDifficultyAdjustment": 1000, "blocksUntilHalving": 513255 }
GET /block/template

Get block template for mining (for mining pools).

Query Parameters

Name Type Description
miner string Miner address to receive reward
POST /mining/submit

Submit mined block solution (for mining pools).

Request Body

{ "height": 12345, "nonce": 987654321, "minerAddress": "NOGO00...", "extraNonce": "optional" }

Response

{ "accepted": true, "reward": 5000000000 }
POST /mine/once

Mine a single block immediately (for testing).

Authentication Required

Requires X-Admin-Token header.

Wallet

GET /wallet/balance/{address}

Get wallet balance for an address.

POST /wallet/sign

Sign a message with private key.

Request Body

{ "address": "NOGO00...", "password": "secure_password", "message": "message_to_sign" }
POST /wallet/sign_tx

Sign a transaction with private key.

GET /wallet/verify

Verify an address format.

GET /wallet/derive

Derive address from seed and index.

Query Parameters

Name Type Description
seed string BIP39 seed phrase
index uint32 Derivation index
GET /wallet/addresses

Derive multiple addresses from seed.

Query Parameters

Name Type Description
seed string BIP39 seed phrase
start uint32 Start index
count uint32 Number of addresses

Network

GET /p2p/getaddr

Get list of known peer addresses.

Response Example

{ "peers": [ { "address": "103.112.185.5:9090", "lastSeen": 1234567890, "services": ["full_node", "miner"] } ] }
POST /p2p/addr

Announce peer addresses to the network.

Community Proposals

GET /api/proposals

Get all community governance proposals.

Query Parameters

Name Type Description
status string Filter by status: active, passed, rejected, completed
GET /api/proposals/{id}

Get details of a specific proposal.

POST /api/proposals/create

Create a new community proposal.

Authentication Required

Requires signed transaction with minimum deposit.

POST /api/proposals/vote

Vote on an active proposal.

Request Body

{ "proposalId": 1, "vote": "yes", "voterAddress": "NOGO00...", "signature": "base64..." }
POST /api/proposals/deposit

Create a deposit for proposal submission.

Utilities

GET /health

Health check endpoint for monitoring.

Response

{ "status": "ok" }
GET /audit/chain

Audit entire blockchain integrity (admin only).

Authentication Required

Requires X-Admin-Token header.

Error Codes

⚠️ Standard Error Response Format

{ "error": { "code": "TX_INVALID_SIGNATURE", "message": "Transaction signature verification failed", "details": {...} } }
Code HTTP Status Description
TX_INVALID_SIGNATURE 400 Transaction signature verification failed
TX_INSUFFICIENT_BALANCE 400 Account has insufficient balance
TX_NONCE_TOO_LOW 400 Transaction nonce is too low (replay)
BLOCK_INVALID 400 Block validation failed
BLOCK_ORPHAN 400 Block is orphaned (missing parent)
NOT_FOUND 404 Resource not found
UNAUTHORIZED 401 Authentication required
INTERNAL_ERROR 500 Internal server error
