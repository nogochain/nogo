## 主网同步节点（推荐）

适用于：仅同步主网数据，不参与挖矿的节点。

### 1. 环境准备

```bash
# 系统要求
- 操作系统：Windows 10+ / Linux (Ubuntu 20.04+) / macOS
- Go 版本：1.21.5+（精确版本）
- 内存：最低 2GB，推荐 4GB+
- 存储：最低 10GB SSD
- 网络：宽带上传/下载 >= 10Mbps
```

### 2. 编译代码

```bash
# 克隆仓库
git clone https://github.com/nogochain/nogo.git
cd nogo/blockchain

# 生产环境编译
go build -race -ldflags="-s -w" -o blockchain.exe .

# 验证编译产物
ls -lh blockchain.exe
```

### 3. 配置环境变量

**Windows PowerShell (推荐)**:

```powershell
# 在项目根目录 (D:\NogoChain) 执行
$env:AUTO_MINE="false"
$env:GENESIS_PATH="nogo/genesis/mainnet.json"
$env:ADMIN_TOKEN="test123"
$env:P2P_PEERS="main.nogochain.org:9090"
$env:SYNC_ENABLE="true"
```

**Linux/macOS**:

```bash
# 在项目根目录执行
export AUTO_MINE="false"
export GENESIS_PATH="nogo/genesis/mainnet.json"
export ADMIN_TOKEN="your_admin_token"
export P2P_PEERS="main.nogochain.org:9090"
export SYNC_ENABLE="true"
```

**可选配置**:
```powershell
# 性能调优（可选）
$env:RATE_LIMIT_REQUESTS="100"      # 每秒请求数限制
$env:RATE_LIMIT_BURST="50"          # 突发请求数限制
$env:TRUST_PROXY="true"             # 信任反向代理
```

### 4. 启动节点

**Windows PowerShell**:
```powershell
# 在项目根目录执行
.\nogo\blockchain\nogo.exe server
```

**Linux/macOS**:
```bash
# 在项目根目录执行
cd nogo/blockchain
./blockchain server
```

**预期日志**:
```
[INFO] NogoChain node listening on :8080 (miner=, aiAuditor=false)
[INFO] P2P listening on :9090
```

### 5. 验证同步状态

```bash
# 检查节点健康状态
curl http://localhost:8080/health

# 查看链信息（高度、难度、rulesHash 等）
curl http://localhost:8080/chain/info | jq

# 预期输出示例：
# {
#   "height": 105,
#   "difficultyBits": 11,
#   "rulesHash": "ea16ad1d4e569580dd495826412b0a89a3733bdc921f9ec6e88c6b44c55574f4",
#   "peersCount": 1,
#   ...
# }
```

### 6. 访问区块浏览器

打开浏览器访问：**http://localhost:8080/explorer/**

功能：
- ✅ 实时区块高度和链信息
- ✅ 区块详情（高度、哈希、时间戳、难度、Nonce、矿工等）
- ✅ 交易列表和详情
- ✅ 内存池查看
- ✅ 节点连接状态（PeersCount）
- ✅ 搜索功能（按高度、哈希、地址、交易 ID）

### 7. 停止节点

```bash
# Windows PowerShell
Get-Process | Where-Object {$_.ProcessName -like "*blockchain*"} | Stop-Process -Force

# Linux/macOS
pkill -f blockchain
```

## 主网挖矿节点

适用于：参与主网挖矿，获取区块奖励的节点。

### 1. 基础配置

完成上述"主网同步节点"的步骤 1-3。

### 2. 启用挖矿

**Windows PowerShell**:

```powershell
# 启用自动挖矿
$env:AUTO_MINE = "true"
$env:MINE_INTERVAL_MS = "1000"
$env:MINE_FORCE_EMPTY_BLOCKS = "true"
```

**Linux/macOS**:

```bash
# 启用自动挖矿
export AUTO_MINE="true"
export MINE_INTERVAL_MS="1000"
export MINE_FORCE_EMPTY_BLOCKS="true"
```

### 3. 启动挖矿节点

```bash
./blockchain.exe server
```

### 4. 监控挖矿状态

```bash
# 查看当前难度和挖矿收益
curl http://localhost:8080/chain/info | jq '.difficultyBits, .currentReward'

# 查看矿工地址余额
curl http://localhost:8080/balance/NOGO006f44f4319250563c65919062932cc1cd7bae04045c355bf53bcb9d7f785c0b473fabfd7c | jq
```

## 常见问题

### 1. 端口被占用

**错误**: `bind: Only one usage of each socket address`

**解决**:
```bash
# Windows - 停止旧节点
Get-Process | Where-Object {$_.ProcessName -like "*blockchain*"} | Stop-Process -Force

# Linux/macOS
pkill -f blockchain

# 等待 2 秒后重新启动
sleep 2
./blockchain.exe server
```

### 2. 同步停止在区块 2

**原因**: PoW 验证算法与主网不一致

**解决**: 使用最新代码，已修复验证逻辑：
- 难度范围验证：`MinDifficultyBits <= difficulty <= MaxDifficultyBits`
- PoW Seal：信任主网（与 Bitcoin SPV 相同）

### 3. 无法连接主网

**检查**:
```bash
# 1. 检查 P2P 端口是否监听
netstat -an | grep 9090

# 2. 检查主网节点是否可达
ping main.nogochain.org
telnet 223.254.144.158 9090

# 3. 查看节点日志
# 控制台输出应显示 "P2P listening on ..." 和 "NogoChain node listening on ..."
```

### 4. RulesHash 不匹配

**错误**: `consensus params mismatch: stored rulesHash=xxx current rulesHash=yyy`

**解决**:
```bash
# 删除本地区块数据，重新同步
rm -rf ./data/
./blockchain.exe server
```

### 5. 浏览器显示 404

**解决**: 确保访问正确的 URL：
- ✅ `http://localhost:8080/explorer/`（注意末尾斜杠）
- ❌ `http://localhost:8080/explorer`

## 配置参数说明

| 参数 | 说明 | 默认值 | 示例 |
|------|------|--------|------|
| `MINER_ADDRESS` | 矿工地址（接收区块奖励） | 空 | `NOGO006f...` |
| `GENESIS_PATH` | 创世文件路径 | `genesis/mainnet.json` | `nogo/blockchain/genesis/mainnet.json` |
| `ADMIN_TOKEN` | 管理 API 认证令牌 | 空 | `6M8BFbyqrEALjn0cw4SmNTUCdZJ7vlz1` |
| `HTTP_ADDR` | HTTP 服务监听地址 | `0.0.0.0:8080` | `0.0.0.0:8080` |
| `P2P_LISTEN_ADDR` | P2P 监听地址 | `0.0.0.0:9090` | `0.0.0.0:9091` |
| `P2P_PEERS` | 初始 P2P 节点列表 | 空 | `223.254.144.158:9090` |
| `NODE_ID` | 当前节点 ID | 自动生成 | `local-node` |
| `AUTO_MINE` | 是否自动挖矿 | `false` | `true`/`false` |
| `MINE_INTERVAL_MS` | 挖矿间隔（毫秒） | `1000` | `1000` |
| `MINE_FORCE_EMPTY_BLOCKS` | 是否挖空区块 | `false` | `true`/`false` |
